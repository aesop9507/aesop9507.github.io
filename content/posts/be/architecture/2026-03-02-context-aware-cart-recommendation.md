## 왜 장바구니 추천은 ‘유사도’만으로 실패하는가

커머스에서 장바구니 추천은 단순히 “비슷한 상품”을 보여주는 문제가 아닙니다. 핵심은 **지금 고객이 어떤 구매 의도로 장바구니를 구성하고 있는지**를 읽어내고, 그 맥락에 맞는 다음 상품을 제안하는 것입니다.

초기 추천 모델은 구현이 빠른 Item2Vec(주문 내 동시 출현 기반 임베딩 유사도) 방식이었습니다. 이 방식은 도입이 쉽지만, 운영 단계에서 아래 두 가지 구조적 한계가 드러납니다.

1. **대체재 편향**
   - 우유를 담으면 시리얼·빵 같은 보완재 대신 다른 우유만 반복 노출
   - 결과적으로 추천 다양성과 크로스셀링 성과 저하

2. **시퀀스 맥락 부재**
   - 동일 상품 집합이라도 담는 순서가 다르면 의도가 다른데, Item2Vec은 이를 구분하지 못함
   - 예: `[라면, 삼겹살, 상추]` vs `[삼겹살, 상추, 라면]`

즉, “무엇을 샀는가”만으로는 부족하고, **“어떤 흐름으로 담았는가”**를 모델링해야 합니다.

## 개선 전략: 2-Stage 추천 파이프라인

### Stage 1) 그래프 기반 임베딩 (Node2Vec)

주문 데이터를 그래프로 구성해 상품/카테고리 관계를 학습합니다.

- Node: 상품, 카테고리
- Edge:
  - Item-Item (함께 구매 연관)
  - Item-Category (콜드스타트 보완)
- 기법:
  - Weighted Random Walk로 시퀀스 생성
  - Node2Vec 학습으로 상품·카테고리 임베딩 생성

#### 왜 이 단계가 중요한가

롱테일 상품은 주문 데이터가 희소해 순수 시퀀스 모델만으로는 학습이 불안정합니다. 그래프 임베딩은 **구조적 연관성**을 먼저 안정화해 다음 단계(Transformer)의 입력 품질을 높여줍니다.

### Stage 2) Transformer 기반 Next-Item Prediction

장바구니를 시계열 시퀀스로 보고 다음 상품을 예측합니다.

- 입력: 과거 장바구니 시퀀스
- 출력: 다음 상품(+카테고리) 예측
- 핵심 구성:
  - Causal Masked Self-Attention (미래 정보 누수 방지)
  - 상품 임베딩 + 카테고리 임베딩 + 위치 임베딩 결합
  - Multi-task learning (상품/카테고리 동시 학습)
  - Focal Loss (인기 상품 쏠림 완화)

#### 실무적으로 좋은 포인트

이 접근은 “정확도”뿐 아니라 “추천 성격”을 바꿉니다.
- 유사 상품 반복 → 보완 상품 추천
- 대체재 중심 → 맥락 중심
- 클릭 유도 → 실제 구매 전환 강화

## 성능 결과가 의미하는 것

사례에서 보고된 개선은 단순 오프라인 점수 상승이 아닙니다.

- 오프라인: HitRate@10 대폭 개선, 카테고리 다양성 개선
- 온라인 A/B:
  - 전체 담기율 증가
  - 바로담기율 증가
  - 주문율 증가

여기서 특히 중요한 신호는 **바로담기율과 주문율 동반 상승**입니다.

이는 “추천이 더 눈에 띄었다” 수준이 아니라,
**고객이 별도 탐색 없이도 추천을 신뢰하고 구매 결정으로 이어갔다**는 뜻입니다.

---

## V2 고도화에서 배울 수 있는 운영 인사이트

초기 성공 이후에도 추가 개선이 가능했던 이유는 모델 아키텍처보다 **데이터 분포와 학습 절차**를 더 정교하게 다뤘기 때문입니다.

### 1) 셀러 타입 분리 학습

마트/편의점은 SKU 구조와 구매 패턴이 다릅니다.
이를 하나의 임베딩 공간에 강제로 넣으면 표현이 흐려집니다.

- 도메인 분리 임베딩 → 맥락 해상도 개선
- 특히 마트처럼 카테고리 복잡도가 높은 도메인에서 효과 큼

### 2) 그래프 연결 품질 개선

- Category-Category edge 추가로 탐색 경로 풍부화
- Bayesian smoothing으로 희소 edge 노이즈 완화

즉, “연결을 늘리는 것”보다 **신뢰도 있는 연결을 만드는 것**이 중요합니다.

### 3) Epoch-wise target sampling

고정 타깃 학습은 시퀀스 일반화에 약합니다.
Epoch마다 타깃을 바꾸면 같은 주문에서도 다양한 prefix/target 조합을 학습해 일반화 성능이 좋아집니다.

## 우리 팀(백엔드/추천 시스템) 관점에서의 적용 체크리스트

1. **오프라인 지표만 보지 말 것**
   - HitRate@K + 다양성 + 커버리지 + 장기 구매 기여도 함께 추적

2. **온라인 지표를 단계별로 분리할 것**
   - 노출 → 담기 → 바로담기 → 주문 전환 퍼널로 관찰

3. **도메인 분리 기준을 먼저 정의할 것**
   - 채널/카테고리/셀러 타입 분리 학습이 필요한지 사전 검증

4. **콜드스타트는 모델이 아니라 데이터 구조 문제로 먼저 접근할 것**
   - 카테고리·브랜드·속성 그래프를 통해 priors 제공

5. **학습 절차의 재현성 확보**
   - 샘플링 로직, seed 정책, 데이터 스냅샷 버전 고정

## 결론

장바구니 추천의 성패는 “비슷한 상품을 얼마나 잘 찾는가”가 아니라,
**고객의 현재 구매 흐름을 얼마나 정확히 읽어내는가**에 달려 있습니다.

이번 사례가 주는 가장 큰 교훈은 다음 한 줄로 정리됩니다.

> 추천 시스템은 모델 교체만으로 좋아지지 않는다. 
> **데이터 구조(그래프), 시퀀스 맥락(Transformer), 운영 실험(A/B)이 함께 설계될 때 성능이 올라간다.**

추천이 클릭을 늘리는 수준을 넘어 실제 주문 전환까지 견인하려면,
앞으로는 “대체재 유사도”보다 “구매 의도 맥락”을 중심에 두는 설계가 기본값이 되어야 합니다.
