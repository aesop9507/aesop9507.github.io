## 개요

전통적인 피싱 사이트는 인기 있는 서비스의 로그인 페이지를 정적으로 복사한 사이트였습니다. 하지만 새로운 피싱-as-a-Service(PhaaS)인 **Starkiller**는 이 패러다임을 완전히 바꾸어 놓았습니다.

Starkiller는 타겟 브랜드의 **실제 로그인 페이지를 동적으로 로드**하고, 사용자 입력을 공격자 인프라를 통해 **실제 사이트로 전달(MITM Reverse Proxy)**합니다. 그리고 가장 놀라운 점은 이 방식으로 **다중 인증(MFA)까지 우회**한다는 것입니다.

Krebs on Security와 Abnormal AI의 분석을 바탕으로, Starkiller의 작동 원리와 이것이 보안에 미치는 영향을 정리합니다.

---

## 1. Starkiller이란 무엇인가?

### 1.1 기존 피싱과의 차이점

| 특징 | 기존 피싱 | Starkiller |
|------|----------|------------|
| 페이지 로드 | 정적 HTML 복사 | 실제 사이트 동적 로드 |
| MFA 우회 | 어려움 (코드 입력 불가) | **가능 (실시간 릴레이)** |
| 탐지 회피 | 도메인 블록리스트 취약 | 정적 페이지 분석 취약 |
| 기술적 난이도 | 높음 | 매우 낮음 |

Starkiller는 피싱 공격의 **모든 기술적 장벽을 제거**하고, "상업용 SaaS 플랫폼"과 유사한 경험을 제공합니다.

### 1.2 제공 서비스

Starkiller는 다음과 같은 기능을 제공합니다:

- **브랜드 선택**: Apple, Facebook, Google, Microsoft 등 인기 서비스 선택
- **URL 마스킹**: `login.microsoft.com@[악성 URL]` 형식의 속임 URL 생성
- **실시간 세션 모니터링**: 타겟 화면 라이브 스트리밍
- **키로거 캡처**: 모든 키스트로크, 쿠키, 세션 토큰 탈취
- **지오트래킹**: 타겟의 위치 추적
- **Telegram 알림**: 새로운 자격증명 획득 시 자동 알림
- **캠페인 분석**: 방문 수, 전환율, 성능 그래프 대시보드

## 2. 작동 원리: 어떻게 MFA를 우회하는가?

### 2.1 공격 체인

```
┌─────────────┐     1. 속임 URL 클릭      ┌──────────────────────┐
│  피해자    │ ────────────────────────→  │  공격자 인프라       │
│  (브라우저) │                         │  (Docker Container) │
└─────────────┘                         └──────────────────────┘
      ↑                                          │
      │ 5. 실제 사이트 응답 반환               │ 2. Headless Chrome
      │                                       │    실제 로그인 페이지 로드
      └───────────────────────────────────────┘
                         │
                         │ 3. 사용자 입력 (ID, PW, MFA)
                         ↓
                 ┌──────────────────┐
                 │  실제 사이트      │ ←─ 4. 공격자가 입력 전달
                 │  (Microsoft 등)  │
                 └──────────────────┘
```

### 2.2 URL 마스킹 기법

Starkiller는 다음과 같이 보이는 URL을 생성합니다:

```
https://login.microsoft.com@malicious[.]ru/xyz
```

여기서 `@` 기호 뒤에 있는 것이 실제 방문 URL입니다. 사용자는 `login.microsoft.com`을 보고 믿게 됩니다.

### 2.3 Docker + Headless Chrome

1. **Docker 컨테이너 생성**: 공격자는 맞춤형 Docker 컨테이너를 생성
2. **Headless Chrome 실행**: 실제 로그인 페이지를 렌더링
3. **리버스 프록시**: 사용자 입력 → 공격자 인프라 → 실제 사이트 → 응답 → 공격자 인프라 → 사용자
4. **세션 하이재킹**: 쿠키와 토큰을 캡처하여 계정 장악

**모든 것이 실제 사이트에서 발생**하기 때문에, 사용자는 자신이 실제 사이트에 로그인했다고 생각합니다.

## 3. 왜 이것이 중요한가?

### 3.1 MFA 우회의 의미

MFA는 보안의 **골드 스탠다드**로 간주됩니다. 하지만 Starkiller는 다음과 같은 방식으로 MFA를 무력화합니다:

> "공격자가 전체 인증 플로우를 실시간으로 릴레이하면, MFA 보호는 설계된 대로 기능하더라도 효과적으로 무력화될 수 있습니다."

- 피해자는 **실제 MFA 페이지**에서 코드를 입력합니다
- 입력된 코드는 공격자 인프라를 통해 실제 사이트로 전달됩니다
- 실제 사이트는 정상적인 로그인으로 인식하고 세션을 생성합니다
- 공격자는 생성된 세션 쿠키와 토큰을 캡처합니다

**결과**: MFA가 완벽하게 작동했음에도 계정이 탈취됩니다.

### 3.2 진화하는 피싱 위협

| 시대 | 피싱 기법 | 탐지 방법 |
|------|----------|----------|
| 과거 | 정적 HTML 복사 | 도메인 블록리스트, SSL 인증서 |
| 최근 | 템플릿 기반 피싱 킷 | 정적 페이지 분석 |
| **현재** | **Starkiller (리버스 프록시)** | 동적 트래픽 분석 필요 |

**기존 탐지 방법의 한계**:

1. **도메인 블록리스트**: 실제 사이트가 로드되기 때문에 무용지물
2. **정적 페이지 분석**: HTML이 실제 사이트와 동일
3. **SSL 인증서**: 실제 사이트의 인증서 사용

### 3.3 진입 장벽의 완전 제거

> "Starkiller는 저숙련 사이버 범죄자들에게 이전에는 도달할 수 없었던 공격 능력을 제공합니다."

기존 피싱 킷 사용에 필요한 기술:
- 서버 구성
- 도메인 관리
- SSL 인증서 발급
- 프록시 서비스 설정

Starkiller는 이 모든 것을 **SaaS 형태로 자동화**했습니다. 초심자도 몇 번의 클릭만으로 전문적인 피싱 캠페인을 시작할 수 있습니다.

## 4. Jinkusu: 엔터프라이즈형 사이버 범죄

Starkiller는 사이버 범죄 그룹 **Jinkusu**가 운영하는 여러 서비스 중 하나입니다:

- 활성 사용자 포럼: 기법 공유, 기능 요청, 문제 해결
- 이메일/연락처 하베스팅: 후속 피싱 캠페인용 타겟 리스트 구축

이는 사이버 범죄의 **상업화 및 기업화**를 보여주는 명확한 사례입니다.

## 5. 방어 전략

### 5.1 기술적 방어

| 방어 계층 | 조치 |
|----------|------|
| **네트워크** | DNS 레벨에서 알려진 악성 도메인 차단 |
| **엔드포인트** | 안티 피싱 솔루션 (URL 분석, 행동 탐지) |
| **인증** | FIDO2/WebAuthn 강화 (물리적 키) |
| **세션** | 지리적 위치 기반 이상 탐지, 세션 타임아웃 |

### 5.2 사용자 교육

**경고 신호 인식**:

1. **URL 확인**: `@` 기호 사용 확인 (예: `microsoft.com@malicious[.]ru`)
2. **예상치 못한 이메일**: 로그인 요청을 보낸 적 없는 서비스
3. **의심스러운 링크**: 짧은 URL, 클릭하기 전 URL 미리보기 사용

### 5.3 조직 정책

- MFA 필수 정책 (가능하면 FIDO2/WebAuthn)
- 정기적인 보안 인식 교육
- 피싱 시뮬레이션 및 피드백
- 의심스러운 계정 활동 모니터링

---

## 6. 결론

Starkiller는 피싱 공격의 새로운 장을 열었습니다. 기존 보안 통제가 무력화되고, 진입 장벽이 제거되면서 사이버 범죄의 상업화가 가속화되고 있습니다.

**핵심 교훈**:

1. MFA만으로는 충분하지 않습니다
2. 행동 기반 탐지가 필수적입니다
3. 사용자 교육의 중요성이 더욱 커졌습니다
4. 사이버 범죄는 기업화되고 SaaS화되고 있습니다

보안은 **계층적 접근(Defense in Depth)**이 필수적입니다. 단일 기술로 모든 위협을 막을 수 없다는 현실을 받아들이고, 여러 방어 계층을 구축해야 합니다.
