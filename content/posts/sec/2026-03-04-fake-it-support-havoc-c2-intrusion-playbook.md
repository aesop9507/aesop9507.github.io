## 왜 이 이슈가 중요한가

2026년 3월 초 공개된 사례에서 공격자는 **이메일 스팸 폭탄 + IT 헬프데스크 사칭 전화**를 결합해 사용자 신뢰를 먼저 무너뜨린 뒤, Havoc C2를 배포하고 내부망으로 빠르게 확산했습니다. Huntress 관측 기준으로는 한 조직에서 **초기 침투 후 11시간 내 9개 엔드포인트 추가 장악**이 이뤄졌습니다.

핵심은 기술 난이도가 아니라, **“사람-프로세스-도구”를 동시에 속이는 운영형 공격**이라는 점입니다.

---

## 공격 체인 요약 (Kill Chain 관점)

### 1) 초기 혼란 조성: 이메일 스팸 폭탄
- 대량 스팸으로 사용자의 메일함을 마비시켜 정상 판단력을 떨어뜨림
- 이후 들어오는 "IT 지원" 연락을 문제 해결 채널로 오인하게 유도

### 2) 신뢰 확보: 가짜 IT 지원 전화 + 원격지원 유도
- 공격자가 IT 헬프데스크를 사칭해 전화
- Quick Assist, AnyDesk 등 합법 원격지원 도구 설치/세션 허용 유도

### 3) 인증정보 수집: Microsoft 유사 페이지 악용
- 원격 세션 중 브라우저를 열어 Microsoft를 가장한 피싱 랜딩 페이지 접근
- 이메일 입력 + "규칙 업데이트" 버튼 클릭을 유도하고 패스워드 오버레이로 자격증명 탈취

### 4) 실행/은닉: 정상 바이너리 + DLL 사이드로딩
- `ADNotificationManager.exe` 등 정상 실행 파일을 악용해 악성 DLL 로드
- 악성 DLL은 Havoc Demon 셸코드를 스레드 형태로 실행

### 5) 탐지 회피: EDR 우회 기법 결합
- 제어 흐름 난독화, 타이밍 지연 루프
- Hell's Gate / Halo's Gate 계열 기법으로 사용자 영역 훅 우회 시도

### 6) 확산/지속성: 스케줄 태스크 + 합법 RMM 병행
- 재부팅 시 자동 실행되도록 스케줄 태스크 등록
- 일부 호스트에서는 Havoc 대신 Level RMM, XEOX 등 합법 원격관리 도구로 백업 지속성 확보

## 보안팀 관점에서 진짜 위험 포인트 4가지

## 1. "사칭 콜 + 원격지원"은 MFA만으로 막기 어렵다
MFA는 로그인 단계를 강화하지만, 사용자가 원격 세션 자체를 열어주면 공격자는 **사용자 컨텍스트 내부**에서 행동합니다. 즉, 계정보안만으로는 불충분하고 **헬프데스크 운영 절차 보안**이 필요합니다.

## 2. "정상 도구 악용" 때문에 전통 시그니처 탐지가 약해진다
AnyDesk, Quick Assist, 정상 바이너리, 합법 RMM 등은 모두 정상 환경에서도 사용됩니다. 따라서 단순 실행 여부가 아니라 **누가, 언제, 어떤 맥락에서 사용했는지**를 행위 기반으로 봐야 합니다.

## 3. 침투 후 확산 속도가 매우 빠르다
11시간 내 다수 단말 확산은 대응팀에 "탐지 후 분석"만으로는 늦다는 메시지입니다. 침해 의심 순간 **선차단·선격리** 가능한 자동화가 없으면 MTTR이 급격히 악화됩니다.

## 4. 공격자 경제성(ROI)이 높다
복잡한 0-day 대신 사회공학 + 공개 C2 + 합법 도구 조합으로 충분히 고도 공격을 구현합니다. 즉, 중견 조직도 동일한 위협에 노출됩니다.

---

## 우리 조직에 바로 적용할 실전 방어 가이드

## A. 헬프데스크 "신원 검증 프로토콜" 강제
- 전화/메신저 IT 지원 요청은 **티켓 번호 + 사내 콜백 번호 재확인** 없으면 작업 금지
- 원격지원 시작 전 사용자에게 고정 문구로 검증 절차 안내
- 고위험 작업(보안설정 변경, 스크립트 실행)은 2인 승인

## B. 원격지원 툴 통제 (Allowlist 기반)
- 승인된 원격지원 도구/버전만 실행 허용
- 비승인 원격지원 도구 설치 시 즉시 차단 또는 경고 격리
- 업무시간 외 원격 세션은 기본 차단 + 예외 승인제

## C. 행위 기반 탐지 룰 강화
- 이메일 폭탄 직후 원격지원 실행 + 외부 로그인 페이지 접근 패턴 상관분석
- 정상 바이너리의 비정상 DLL 로딩 이벤트 탐지
- 신규 스케줄 태스크 등록 + 외부 C2 통신 결합 이벤트를 High 이상으로 자동 승격

## D. 자격증명 보호와 세션 통제
- 관리자/고위험 계정은 피싱 저항형 MFA(FIDO2 등) 우선 적용
- 브라우저 기반 인증 단계에 도메인 무결성 점검(기업 보안 확장/프록시)
- 비정상 위치·기기에서의 세션 생성 시 조건부 접근(추가 검증 또는 차단)

## E. 초기 1시간 대응 런북(필수)
1. 의심 단말 네트워크 격리
2. 원격지원 세션 강제 종료
3. 최근 생성된 스케줄 태스크/서비스/시작프로그램 전수 점검
4. 동일 사용자/동일 부서 단말 IOC 확산 탐지
5. 계정 비밀번호/토큰 즉시 회수 및 세션 만료

---

## 운영 KPI 제안 (보안 성숙도 측정)

- **MTTD(탐지시간)**: "스팸 폭탄 시작 → 최초 경보" 15분 이내
- **MTTC(차단시간)**: "최초 경보 → 원격세션 차단" 10분 이내
- **Lateral Spread Rate**: 최초 감염 후 추가 감염 단말 수(시간당)
- **Unauthorized RMM Rate**: 비승인 원격관리 도구 탐지 건수/월
- **Helpdesk Verification Compliance**: 신원검증 절차 준수율 95%+

## 결론

이번 사례는 "악성코드가 정교해졌다"를 넘어, **공격 운영 모델이 사람의 업무 프로세스까지 장악**하고 있음을 보여줍니다.

따라서 방어 전략도 단일 솔루션 중심이 아니라,
1) 헬프데스크 검증 절차,
2) 원격지원 도구 통제,
3) 행위 기반 탐지,
4) 초기 1시간 자동 대응
을 하나의 체계로 묶어야 합니다.

보안팀이 지금 당장 해야 할 일은 새로운 제품 구매보다, **"가짜 IT 지원 시나리오"를 전제로 한 테이블탑 훈련과 운영 룰 정비**입니다.
